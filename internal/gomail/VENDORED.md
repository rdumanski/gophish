# Vendored: gophish/gomail

This package is a verbatim copy of `github.com/gophish/gomail` at commit
[`1f6d0dfd512e`](https://github.com/gophish/gomail/commit/1f6d0dfd512e)
(2020-08-18, the most recent commit at the time of vendoring), with the local
edits noted below.

## Why vendored

`github.com/gophish/gomail` has been dormant since August 2020 and is itself a
fork of `github.com/go-gomail/gomail`, which has been dormant since 2017.
Vendoring removes the supply-chain dependency on a dead repository while
preserving the two extensions Gophish relies on:

- `NewWithDialer` — constructs a `Dialer` backed by a caller-supplied
  `net.Dialer` (used to enforce the SSRF-safe `dialer.Dialer()` for outbound
  SMTP connections).
- `SendCustomFrom` — sends a message with an envelope-from address distinct
  from the header `From:`, used by Gophish to set the campaign tracking
  identifier as the SMTP `MAIL FROM` while keeping the visible sender
  intact.

## Local edits

- **Removed `mime_go14.go`** (Go 1.4 quoted-printable shim, gated by
  `// +build !go1.5`). Gophish targets Go 1.25; the shim was dead code, and
  dropping it also retires the only remaining transitive dependency
  (`gopkg.in/alexcesaro/quotedprintable.v3`, last touched 2015).
- **Dropped the `// +build go1.5` tag from `mime.go`** since it is no longer
  the conditional sibling of anything.
- **Skipped `example_test.go`** (package `gomail_test`, doc-only examples that
  would pull `os.Stdout`-style fixtures with no value to the in-tree tests).
- **Updated `doc.go`** to document the vendoring provenance.
- **Re-formatted `send_test.go`** with `gofmt`: the upstream file had a
  misaligned `const` block (`testAlternativeFrom` was longer than the other
  identifiers but the rest of the block kept the original 1-space gutter)
  and a stray double blank line before `TestSend`. The repo's CI runs
  `gofmt -d .` as a hard gate, so we can't carry the upstream non-canonical
  whitespace without breaking the Build & Test job. Behavior unchanged.

No other source changes were made; method signatures, exported identifiers,
and behavior are byte-identical to the upstream fork. Diffs against future
upstream activity (unlikely) can be regenerated by:

```
diff -ur \
  $(go env GOMODCACHE)/github.com/gophish/gomail@v0.0.0-20200818021916-1f6d0dfd512e/ \
  internal/gomail/
```
